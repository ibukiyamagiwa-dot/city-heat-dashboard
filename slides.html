本編 — ダッシュボード

ビジョン

  • 都市の「にぎわい」「熱狂度」を感覚ではなくデータで説明
  • CITY SONAR 型：非専門家にも追える Web UI
  • スコアの根拠・信頼度・内訳まで見せる
派手なエンタメアプリではない。説明可能性が研究の軸。

解きたい問題

  • 施設数の絶対量だけだと大都市（東京）が有利になりやすい
  • イベント・SNS など「その日の熱量」は未接続
  • OSM は公的統計より信頼度が低い（補助指標として扱う）

→ スコア設計と UI で「見える化して検証する」

問題は3つに絞って話す。駅別・progress.mdは口頭または課題スライドへ。

スコアの考え方

  • 対象：東京・大阪・名古屋・福岡
  • 統合：気象 / 人流 / 文化施設 / 現代エンタメ(OSM) / 人口補正

mania_score = scale_power + accessibility_power + weather_boost

  • scale_power … 絶対規模（人流・施設の底力）
  • accessibility_power … 人口・人流あたりの充実度
  • weather_boost … 当日の気象
二軸の意味を30秒で。大都市バイアスを相対量で見える化したのが研究の核。
