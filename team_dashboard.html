連携一覧

縦連携（管理→実行）

  • strategy_mgmt (統括戦略室マネージャー) → strategy_exec (統括戦略室実行担当) / 全体戦略の管理と実行
  • pmo_mgmt (PMOマネージャー) → pmo_exec (PMO実行担当（秘書）) / 統括管理と進捗可視化
  • planner_mgmt (企画部門マネージャー) → planner_exec (企画部門実行担当) / 企画方針の管理と実行
  • audit_mgmt (監査部門マネージャー) → audit_exec (監査部門実行担当) / 監査基準の管理と現場監査
  • data_mgmt (データ部門マネージャー) → data_exec (データ部門実行担当) / データ運用方針の管理と収集実行
  • dev_mgmt (開発部門マネージャー) → dev_exec (開発部門実行担当) / 開発優先度の管理と実装
  • solution_mgmt (改善解決案部門マネージャー) → solution_exec (改善解決案部門実行担当) / 改善方針の管理と解決案作成
  • feedback_mgmt (フィードバック部門マネージャー) → feedback_exec (フィードバック部門実行担当) / 評価観点の管理とフィードバック収集

横連携（部門間）

  • strategy_exec (統括戦略室実行担当) → pmo_exec (PMO実行担当（秘書）) / 全体方針と進捗管理の同期
  • planner_exec (企画部門実行担当) → data_exec (データ部門実行担当) / 要件とデータ仕様の同期
  • data_exec (データ部門実行担当) → audit_exec (監査部門実行担当) / データ真偽と出典確認
  • audit_exec (監査部門実行担当) → dev_exec (開発部門実行担当) / 監査指摘の実装反映
  • dev_exec (開発部門実行担当) → planner_exec (企画部門実行担当) / 実装制約の企画フィードバック
  • feedback_exec (フィードバック部門実行担当) → solution_exec (改善解決案部門実行担当) / αβ評価を改善案へ変換
  • solution_exec (改善解決案部門実行担当) → planner_exec (企画部門実行担当) / 改善点に対する企画解決案
  • solution_exec (改善解決案部門実行担当) → dev_exec (開発部門実行担当) / 再開発タスクとして実装へ戻す
  • planner_mgmt (企画部門マネージャー) → dev_mgmt (開発部門マネージャー) / スコープと納期の調整
  • audit_mgmt (監査部門マネージャー) → data_mgmt (データ部門マネージャー) / 品質基準とデータ運用基準の整合
  • pmo_mgmt (PMOマネージャー) → planner_mgmt (企画部門マネージャー) / 全体優先度の調整
  • pmo_mgmt (PMOマネージャー) → audit_mgmt (監査部門マネージャー) / 提出判定条件の調整
  • pmo_mgmt (PMOマネージャー) → data_mgmt (データ部門マネージャー) / 運用・法務リスクの調整
  • pmo_mgmt (PMOマネージャー) → dev_mgmt (開発部門マネージャー) / リソース計画の調整

改善ループ（α/β → フィードバック → 解決案 → 再開発）

α版改善ループ

開発部門実行担当dev_exec → フィードバック部門実行担当feedback_exec → 改善解決案部門実行担当solution_exec → 企画部門実行担当planner_exec → データ部門実行担当data_exec → 監査部門実行担当audit_exec → 開発部門実行担当dev_exec

β版改善ループ

開発部門実行担当dev_exec → フィードバック部門実行担当feedback_exec → 改善解決案部門実行担当solution_exec → 開発部門実行担当dev_exec → 監査部門マネージャーaudit_mgmt
