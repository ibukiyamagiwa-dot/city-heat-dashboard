# 進捗ドキュメント

## 1. 決定したアプリ案の概要
選ばれたアプリ案は「トラフィックインサイト」です。このアプリは都市内の交通量を可視化し、ユーザーが混雑する時間帯を避けて効率的に移動できるようサポートします。主要機能として、交通データのリアルタイム表示、混雑度の通知機能などを実装予定です。

## 2. 監査班からの評価コメント
- **実現可能性**: 90点
  - 公共交通機関APIやGoogle Maps APIを使用することができ、データ収集が容易。
  - ユーザビリティを考慮した機能を提供。
- **懸念点**: APIの利用制限が開発や運用に影響する可能性がある。
  
## 3. 現在の完了ステータス
- **実施済み項目**:
  - アプリ案の選定
  - 初期のプロトタイピングと機能要件の整理
- **進行中項目**:
  - 必要なデータソースとの連携準備

## 4. 次のステップ（ToDoリスト）
- [ ] 公共交通機関API及びGoogle Maps APIの申請と設定
- [ ] 交通データのリアルタイム取得機能の実装
- [ ] ユーザーテスト用の初期画面設計
- [ ] 混雑度通知機能の開発
- [ ] ヒートマップの表示機能の開発

## 5. 実現可能性を高めるための注意点（データ・法務・開発工数）
- **データ**:
  - APIの利用規約を確認し、過剰収集を避ける。
- **法務**:
  - 個人情報を含まないデータの使用を徹底する。
- **開発工数**:
  - 各機能の開発スケジュールを明確にし、人員配置を工夫することが重要。
