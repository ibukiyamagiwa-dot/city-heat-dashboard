# 都市熱狂度スコア設計 v0.5

## 現在のスコア式

都市熱狂度は、都市の絶対的な規模、人口・人流あたりの施設充実度、その日の気象による一時的な追い風・向かい風を分けて評価する。

```text
heat_score = scale_power + accessibility_power + weather_boost
```

```text
scale_power = 35.0 + flow_bonus + culture_venue_bonus + modern_entertainment_bonus
accessibility_power = culture_per_population_bonus + modern_per_population_bonus + culture_per_flow_bonus
weather_boost = temp_score - precip_penalty - wind_penalty
```

## モデル拡張の理由

従来のモデルは、主要駅利用者数や施設数の絶対量を重視していた。そのため、東京のような大都市は人口・人流・施設数の母数が大きく、単純な施設数比較では有利になりやすい。

教授からの指摘である「施設数だけだと東京が勝ちやすい。人口や有効的な人数に対する施設数で見ないと説得力が弱い」を受け、今回のモデルでは次の2つを分けて評価する。

- `scale_power`
  - 人流、文化施設数、現代エンタメ施設数などの絶対量。
  - 「都市としてどれだけ大きな受け皿を持つか」を示す。

- `accessibility_power`
  - 人口あたり・駅利用者あたりの施設充実度。
  - 「都市規模に対して、施設がどれくらい身近に存在するか」を示す。

これにより、「東京が強いのは単に大きいからなのか」「小規模でも人口あたりでは充実している都市があるのか」という問いに答えられる。卒論上は、絶対量と相対量を併記することで、都市規模バイアスを完全に消すのではなく、見える化して検証可能にする狙いがある。

`weather_boost` は従来どおり、気温・降水量・風速など日々変動する要因を別枠で扱う。これにより、構造的な強さと当日の条件を混同しない。

## 構成要素

- `flow_bonus`
  - 出典: 国土交通省 国土数値情報 駅別乗降客数
  - 信頼度: medium
  - 算出: `station_users` を4都市間で正規化し、0.0〜18.0点に変換
  - 意味: 主要駅利用者が多い都市ほど、人流の底力が高い

- `culture_venue_bonus`
  - 出典: 文部科学省 社会教育調査（劇場・音楽堂）
  - 信頼度: medium
  - 算出: `event_venues` を4都市間で正規化し、0.0〜7.0点に変換
  - 意味: 劇場・音楽堂を「現代エンタメ全体」ではなく、公的統計に基づく文化施設指標として扱う

- `modern_entertainment_bonus`
  - 出典: OpenStreetMap / Overpass API
  - 信頼度: low
  - 対象例: 映画館、劇場、博物館、美術館、スタジアム、テーマパーク、クラブ、アートセンター、スポーツセンター
  - 算出: 都市中心から半径15km以内の施設数を4都市間で正規化し、0.0〜8.0点に変換
  - 意味: 文科省の劇場・音楽堂だけでは捉えきれない、現代的な余暇・娯楽施設の厚みを補助的に評価する

- `accessibility_power`
  - 出典: 総務省人口推計、国土数値情報駅利用者数、文科省社会教育調査、OSM
  - 信頼度: medium〜low
  - 使用指標:
    - `culture_venues_per_100k` = 文化施設数 / 人口 * 100,000
    - `modern_entertainment_per_100k` = 現代エンタメ施設数 / 人口 * 100,000
    - `culture_venues_per_million_station_users` = 文化施設数 / 駅利用者数 * 1,000,000
  - 意味: 絶対数だけではなく、都市規模に対して施設がどれだけ充実しているかを見る

- `temp_score`
  - 出典: Open-Meteo
  - 信頼度: high
  - 算出: 気温を0.0〜18.0点に変換
  - 意味: 気温が活動しやすいほど今日のブーストが上がる

- `precip_penalty` / `wind_penalty`
  - 出典: Open-Meteo
  - 信頼度: high
  - 意味: 降水・強風は屋外活動を下げる可能性があるため減点

## UI上の説明

画面では、総合スコアだけでなく以下を表示する。

- 順位
- `scale_power`（絶対量の規模パワー）
- `accessibility_power`（人口・人流あたりの充実度）
- `weather_boost`（今日のブースト）
- 強み（例: 主要駅利用者数が4都市中1位）
- 弱み（例: 人口あたり文化施設数が4都市中最少）

これにより、点数差から「なぜ高いのか」「なぜ低いのか」に加えて、「大都市だから強いだけなのか」「人口あたりではどうなのか」という問いを自然に確認できるようにする。

## 公的統計とOSMの信頼度の違い

公的統計（国土数値情報、文科省社会教育調査、総務省人口推計）は、調査方法や基準年が明示されており、卒論で根拠として説明しやすい。一方で、都道府県単位である、調査年が少し古い、開催数ではなく施設数であるといった限界がある。

OpenStreetMapは、映画館・テーマパーク・クラブなど現代エンタメに近い施設種別を柔軟に取得できる利点がある。しかし、登録状況が地域ごとの編集密度に依存し、タグ表記にも揺れがある。そのため、本研究ではOSMを高信頼の公的統計と同列には扱わず、`modern_entertainment_bonus` として低信頼の補助指標に位置づける。

この設計により、「信頼できる公的統計で土台を作り、OSMで現代エンタメの不足部分を補う」という役割分担を明確にする。

## 廃止したもの

- 固定 `event_bonus=12.0`（仮説値）→ 廃止
- `event_bonus` 合成フィールド → 廃止（底力と気象を分離）
- `venue_bonus` という曖昧な名称 → `culture_venue_bonus` に再定義
